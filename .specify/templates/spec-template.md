### Functional Requirements

- **FR-001**: System MUST [specific capability, e.g., "allow users to create accounts"]
- **FR-002**: System MUST [specific capability, e.g., "validate email addresses"]
- **FR-003**: Users MUST be able to [key interaction, e.g., "reset their password"]
- **FR-004**: System MUST [data requirement, e.g., "persist user preferences"]
- **FR-005**: System MUST [behavior, e.g., "log all security events"]
- **FR-MCP-001**: Each MCP tool MUST define its name, input schema, output shape, and error responses.
- **FR-VAULT-001**: Vault reads and writes MUST remain local unless explicit opt-in external behavior is specified.
- **FR-WRITE-001**: Append operations MUST preserve existing note content and add only the requested change note.

### Key Entities *(include if feature involves data)*

- **MCP tool**: User-invoked capability exposed to an MCP client; includes name, schema, output, and error model.
- **Vault note**: Markdown file in the Obsidian vault; identified by path and metadata available through Obsidian APIs.
- **Change note**: Text appended to an existing vault note by explicit MCP request; includes target note and appended content.
- **Plugin settings**: Persisted configuration for consent, feature enablement, and user preferences.
